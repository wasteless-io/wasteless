Connect your AWS account

About 10 minutes, read-only by default. Wasteless cannot modify anything unless you later create the optional remediation role.

Read-only by default

The role grants Describe / Get / List only. No permission to create, change, or delete anything in your account.

Your resources stay private

It reads configuration and metrics, never the data inside your resources, no S3 object contents, database rows, or secret values.

Temporary access

Credentials are assumed in short-lived sessions each run, not a long-lived key sitting in your account.

Revoke anytime

Delete the CloudFormation stack and every permission wasteless has is gone instantly, nothing to clean up by hand.

1

Create the wasteless roles in your AWS account

One CloudFormation stack creates wasteless-readonly , Describe/Get/List permissions only. This is the only part that happens inside your AWS account, and the button below pre-fills it all.

Opens your AWS console on a pre-filled Create stack review page: sign in as an administrator, tick "I acknowledge that AWS CloudFormation might create IAM resources with custom names", click Create stack, wait for CREATE_COMPLETE (~1 min), then come back here and hit Test connection, the fields in step 2 are already filled in. If someone else manages AWS, send them the link with Copy link. Type an External ID in step 2 before opening the link to bake it in.

Manual alternatives (no pre-filled link)
  • CloudFormation (no tooling needed): upload onboarding/cloudformation/wasteless-onboarding.yaml in the AWS console → Create stack
  • Terraform (only if your team already uses it): apply onboarding/terraform/

Full walkthrough: docs/CTO_QUICKSTART.md (short) or docs/AWS_SETUP.md (detailed, per-permission).

Watch the dashboard fill

A first collection starts automatically the moment you save, and the dashboard fills up within a couple of minutes; collection then re-runs on schedule. Nothing in your account is ever modified without an explicit approval in Recommendations.
